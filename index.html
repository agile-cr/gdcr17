Práctica

Reglas básicas

  • Sessión: 45 minutos + 10 minutos retro + 5 min pausa
  • Es necesario hacer Pair-programming, ya que la transferencia de conocimiento contenida en esta actividad es esencial para la práctica.
  • Mejor si se usa Test-Driven Development (TDD)
  • Tras cada sesión, se debe cambiar de pareja
  • Tras cada sesión, se debe borrar el código. Nada de ramas o "stash": no dejar ni rastro.

Conway's Game of Life

En cada instante de tiempo, tienen lugar las siguientes transiciones:

  • Inanición: Una célula con menos de dos vecinos vivos, muere.
  • Sobrepoblación: Una célula con más de tres vecinos vivos muere.
  • Supervivencia: Una célula viva con dos o tres vecinos vivos sobrevive.
  • Generación: Una célula muerta con exactamente tres vecinos vivos renace.

Sessión ¿3?

Contra reloj

  1. Inicializar un repositorio.
  2. Pon una cuenta atrás de 3 minutos.
  3. Refactorizar el código o escribir tests
  4. Si se acaba el tiempo y el refactor/test no está terminado, revertir y volver a empezar
  5. Si el refactor/test se acaba antes de que pase el tiempo, commit.
  6. Reiniciar el tiempo.
  7. Volver al 3.

Session ¿5?

TDD como si te fuera la vida en ello

  1. Escribir un test tan pequeño como sea posible
  2. Comprobar que falla
  3. Hacerlo pasar escribiendo la menor cantidad de código posible EN EL MÉTODO DE TEST.
  4. Refactorizar SOLO para evitar duplicación.
  5. SÓLO se extrae un método si hay suficiente duplicación de código.
  6. SÓLO se extrae una clase si hay un número claro de métodos relacionados.
  7. Escribir el siguiente test (paso #1).

Session ¿6?

  1. Escribir un test
